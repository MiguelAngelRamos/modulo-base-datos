Aquí tienes **5 consultas avanzadas** que utilizan `GROUP BY`, filtros `WHERE`, condiciones `HAVING` y múltiples `JOINs`:

### 1. Ventas totales y promedio por país de origen del cliente

Esta consulta ayuda a entender qué mercado es el más rentable.

```sql
SELECT 
    c.country, 
    COUNT(s.id) AS cantidad_ventas,
    SUM(s.total_amount) AS venta_total,
    ROUND(AVG(s.total_amount), 2) AS promedio_por_venta
FROM electronica_store.customers c
INNER JOIN electronica_store.sales s ON c.id = s.customer_id
GROUP BY c.country
ORDER BY venta_total DESC;

```

### 2. Inventario: Valorización de stock por categoría

Aquí agrupamos para saber cuánto dinero tenemos "detenido" en bodega por cada tipo de producto.

```sql
SELECT 
    cat.name AS categoria,
    COUNT(p.id) AS tipos_de_productos,
    SUM(p.stock) AS unidades_totales,
    SUM(p.stock * p.price) AS valor_inventario_total
FROM electronica_store.categories cat
LEFT JOIN electronica_store.products p ON cat.id = p.category_id
GROUP BY cat.name
ORDER BY valor_inventario_total DESC;

```

### 3. Filtro de Marcas "Premium" (Uso de HAVING)

Queremos ver qué marcas tienen un catálogo cuyo precio mínimo sea superior a $300, lo que indica que no venden productos de gama baja.

```sql
SELECT 
    brand, 
    COUNT(id) AS cantidad_modelos,
    MIN(price) AS precio_entrada,
    MAX(price) AS precio_tope
FROM electronica_store.products
GROUP BY brand
HAVING MIN(price) > 300
ORDER BY precio_entrada ASC;

```

### 4. Ranking de Clientes (Top compradores)

Agrupamos por cliente para identificar a nuestros "Power Users" que han gastado más de $1,000 en total.

```sql
SELECT 
    c.full_name,
    c.email,
    COUNT(s.id) AS nro_compras,
    SUM(s.total_amount) AS total_gastado
FROM electronica_store.customers c
INNER JOIN electronica_store.sales s ON c.id = s.customer_id
GROUP BY c.full_name, c.email
HAVING SUM(s.total_amount) > 1000
ORDER BY total_gastado DESC;

```

### 5. Análisis temporal: Ventas por mes

Aunque los datos que insertamos son de meses variados, esta consulta agrupa por el mes de la venta (usando `EXTRACT`).

```sql
SELECT 
    EXTRACT(MONTH FROM sale_date) AS mes,
    COUNT(*) AS total_transacciones,
    SUM(total_amount) AS recaudacion_mensual
FROM electronica_store.sales
GROUP BY mes
ORDER BY mes;

```

### ¿Qué permiten estas consultas técnicamente?

1. **Funciones de Agregación:** Usamos `SUM` (suma), `AVG` (promedio), `COUNT` (conteo), `MIN` (mínimo) y `MAX` (máximo).
2. **Filtros Post-Agrupado:** El uso de `HAVING` permite filtrar resultados que ya fueron calculados (como el total gastado).
3. **Limpieza de Datos:** Usamos `ROUND` para que los promedios no tengan demasiados decimales.
